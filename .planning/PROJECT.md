# Baqsy System

## What This Is

Платформа автоматизированного бизнес-аудита для предпринимателей Казахстана и СНГ. Клиент проходит онбординг в Telegram-боте, выбирает тариф на сайте, оплачивает через CloudPayments, отвечает на отраслевую анкету (шаблоны подставляются автоматически по отрасли его бизнеса), администратор вручную готовит персональный аудит, а система генерирует именной PDF-отчёт и отправляет клиенту в Telegram и WhatsApp.

Продукт — симбиоз конвейера (автоматическая категоризация, анкеты, оплата, доставка) и ручной экспертизы (аудит пишет человек).

## Core Value

**Клиент платит за персональный PDF-аудит своего бизнеса и получает его автоматически после ручной проверки администратором** — этот путь от первого контакта до доставки отчёта должен работать безотказно.

### Out of Scope

- **AI-генерация аудита** — по требованию заказчика аудит пишет человек, ИИ на v1 не используется
- **Email-канал доставки** — только Telegram + WhatsApp
- **Мобильное приложение** — web + TG достаточно
- **Мультиязычность** — только русский на v1 (KZ/EN отложены)
- **Оплата в USD напрямую** — только KZT через CloudPayments KZ
- **Мультитенантность / white-label** — один экземпляр системы
- **Реал-тайм чат с аудитором** — коммуникация асинхронная, через бота
- **OAuth / соцсети для логина** — админ авторизуется по email/паролю, клиент по telegram_id
- **Свой платёжный шлюз** — только CloudPayments
- **Автоматические возвраты** — возврат обрабатывается вручную админом

## Context

**Заказчик и продукт.** Заказчик — владелец методологии бизнес-аудита, выпускает подарочную книгу «Вечный Иль» в комплекте с отчётом. Требует «солидный» визуальный стиль и полную автономию админки (изменение цен, вопросов, текстов — без разработчика).

**Ключевой риск заказчика — зависимость от одного разработчика.** В ТЗ явно прописано: исходный код должен быть в его репозитории, использоваться популярный стек, оставлена документация, возможность развернуть на другом хостинге за пару часов. Это формирует требования: Docker Compose, README, миграции Django, `.env.example`, бэкапы PG.

**Воронка (согласовано).** Бот → 5 базовых вопросов → сайт/тариф → оплата CloudPayments KZ → возврат в бота → отраслевая анкета из 27 вопросов по одному → уведомление админа → ручной аудит → «Подтвердить» → генерация PDF → доставка в TG + WA.

**Отраслевые шаблоны — ключевая фишка.** В зависимости от отрасли (Ритейл, IT, Производство, Услуги, F&B и т.д.) подставляется свой шаблон из 27 вопросов. Блок А (вопросы 1–8) общий во всех шаблонах — это метаданные для фильтрации статистики в CRM. Блоки Б и В отличаются от отрасли к отрасли.

**Тарифы (в тенге, утверждено).**
- Ashıde 1 — 45 000 ₸ — базовый отчёт на 7–9 параметров
- Ashıde 2 — 135 000 ₸ — расширенный отчёт на 18–24 параметра
- Upsell — 90 000 ₸ — апгрейд с 1 на 2 без повторной анкеты

**Канал доставки клиенту — дубль.** Отчёт отправляется и в Telegram (где проходила анкета), и в WhatsApp через Wazzup24 — это явное требование заказчика.

## Constraints

- **Стек:** Python 3.12 + Django 5 + DRF + PostgreSQL 16 + React 18 + TypeScript — популярный стек, требование заказчика (чтобы легко найти замену разработчику)
- **Бот:** aiogram 3.x — современный async-фреймворк для Telegram
- **Платежи:** только CloudPayments KZ — требование заказчика
- **WhatsApp:** Wazzup24 — согласовано
- **Валюта:** KZT — согласовано
- **PDF:** WeasyPrint (HTML→PDF) для максимальной гибкости вёрстки
- **Автономная админка:** цены, вопросы анкеты, тексты лендинга — всё через UI, без деплоя
- **Версионирование шаблонов:** обязательное — редактирование анкеты не должно ломать старые заказы
- **Идемпотентность webhook'ов:** обязательная — платёжные webhook'и могут прилетать повторно
- **Бэкапы:** автоматические дампы PostgreSQL по cron в MinIO
- **Документация:** README с инструкцией развёртывания на новом хостинге за ≤2 часа
- **Собственность кода:** единый Git-репозиторий, полный контроль
